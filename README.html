Ladder Logic Security Analyzer

Overview

This tool uses graph isomorphism techniques to detect 14 common security vulnerabilities in PLC ladder logic programs. It can analyze ladder logic code, visualize program graphs, and provide specific remediation advice for each vulnerability.

How to Use

  1. Open the index.html file in your web browser
  2. Click "Browse" to upload a ladder logic text file (sample files are provided)
  3. Click "Analyze for Vulnerabilities" to run the security analysis
  4. Review the results summary and click on specific vulnerabilities for details
  5. Each vulnerability includes code snippets and recommended fixes

Sample Files

Two sample files are included for testing:

  • sample_ladder.txt - Contains examples of all 14 vulnerability patterns
  • sample_secure_ladder.txt - Contains secure, remediated versions of the same patterns

Vulnerability Patterns

The analyzer checks for these 14 common vulnerability patterns:

Code Name Description
V-001 Unconditional coil overwrite Ladder logic that unconditionally overwrites an output coil without proper conditions
V-002 Shadow reset (seal-in loop) A circuit that resets a seal-in loop without proper validation
V-003 Timer bomb (preset from writable tag) Timer preset values come from HMI-writable tags without validation
V-004 Unsafe mode override Using MOV 1 to override safety or diagnostic tags
V-005 Monolithic main block Main routine with excessive nodes, making it harder to maintain and secure
V-006 Unvalidated timer/counter preset Timer or counter values without validation logic
V-007 Mutually-exclusive coils asserted Antagonistic coil pairs (e.g., Motor_FWD and Motor_REV) asserted simultaneously
V-008 HMI blind-write HMI writing to PLC variables without validation
V-009 Out-of-bounds indirection Array indexing without bounds checks
V-010 Rapid toggle / missing debounce Paired outputs without debounce timer protection
V-011 Missing plausibility cross-check Missing validation between redundant sensor tags
V-012 No physical-limit clamp PID blocks without physical limits on operator inputs
V-013 Flag-blind math Math operations without status bit checking for alarms
V-014 Alert-trap bypass Alarm coil written from multiple rungs where one lacks conditions

Technical Details

The analyzer uses the following techniques:

  • Parses ladder logic code into a graph representation
  • Uses graph isomorphism to detect vulnerability patterns
  • Provides visual representations using Cytoscape.js
  • Offers detailed remediation advice based on secure PLC coding practices
